## 📖 درباره پروژه

**DX-Proxy** یک سیستم مدیریت اختصاصی، مدرن و پرسرعت برای راه‌اندازی و کنترل پروکسی‌های چند سکرتی تلگرام (MTProto) است که بر پایه‌ی هسته‌ی قدرتمند `mtg-multi` ساخته شده.

نکته‌ی مهم این پنل اینه که پنل وب و منوی خط‌فرمان (ترمینال)، هر دو از یک منبع اطلاعاتی واحد استفاده می‌کنند؛ یعنی هر تغییری که از یک طرف بدید (چه از سایت، چه از ترمینال)، همون لحظه توی طرف دیگه هم اعمال می‌شه و هیچ‌وقت این دو تا با هم ناهماهنگ نمی‌شن.

### ✨ ویژگی‌های کلیدی

| ویژگی | توضیح |
|---|---|
| 🎨 **طراحی شیشه‌ای مدرن** | تم تاریک لوکس، کارت‌های آماری زنده و افکت‌های نوری پویا |
| ⚡ **تشخیص خودکار Railway** | بدون نیاز به پیکربندی دستی؛ دامنه و اطلاعات اتصال به‌صورت خودکار خونده می‌شن |
| 👥 **مدیریت هوشمند کاربران** | ساخت، فعال/غیرفعال‌سازی آنی، تعویض سکرت و حذف کاربر، همراه با تولید QR Code |
| 🔄 **همگام‌سازی ۱۰۰٪** | هماهنگی کامل بین پنل وب و خط‌فرمان داخل کانتینر |
| 🔐 **امنیت بالا** | اجبار به تغییر یوزر/پس پیش‌فرض در همون اولین ورود |

## 🚀 راهنمای گام‌به‌گام راه‌اندازی

### مرحله ۱ — فورک کردن پروژه

اول از همه پروژه رو از گیت‌هاب **فورک** کنید تا یه نسخه از اون توی اکانت گیت‌هاب خودتون داشته باشید.

### مرحله ۲ — اتصال Railway به گیت‌هاب

وارد Railway بشید و اکانت گیت‌هاب خودتونو بهش وصل کنید. بعد از اتصال، پروژه‌ای که فورک کردید رو پیدا کرده و برای دیپلوی انتخاب کنید.

### مرحله ۳ — دیپلوی اولیه و تنظیم Networking

بعد از دیپلوی اول، وارد بخش **Settings** پروژه بشید و برید سراغ قسمت **Networking**. اینجا دو تا گزینه می‌بینید که باید درست بشناسیدشون:

- 🌐 **Generate Domain (اختیاری)**
  اگه دوست دارید پنل مدیریتتون به شکل یه **سایت واقعی** باشه (که بتونید از هر جا باهاش وارد بشید، کاربر اضافه کنید، مصرف رو ببینید و...)، این گزینه رو بزنید. با زدنش، Railway یه دامنه براتون می‌سازه. **حواستون باشه پورت داخلی رو روی `2053` تنظیم کنید.**
  اگه فقط می‌خواید خودتون به‌صورت ساده و بدون پنل تصویری استفاده کنید، لازم نیست این گزینه رو بزنید؛ کاملاً انتخابیه.

- 🔌 **TCP Proxy (اجباری)**
  این گزینه برای وصل شدن کلاینت‌های تلگرام به پروکسی شماست و **حتماً باید فعال بشه**. پورت داخلی رو روی `8080` بذارید.

بعد از تنظیم این دو مورد، همه چیز رو **Save** کنید. با زدن Save، بالای صفحه گزینه‌ی **Deploy** ظاهر می‌شه؛ روش کلیک کنید تا دیپلوی جدید با تنظیمات درست انجام بشه.

### مرحله ۴ — صبر برای سبز شدن دیپلوی

بعد از زدن Deploy، چند لحظه صبر کنید تا وضعیت دیپلوی توی Railway **سبز** بشه. سبز شدن یعنی سرویس شما با موفقیت بالا اومده و آماده‌ی استفاده‌ست.

## 💻 دو روش برای مدیریت پروکسی

بعد از سبز شدن دیپلوی، بسته به نیازتون یکی از این دو روش رو انتخاب کنید:

### روش ۱ — پنل تصویری وب (برای استفاده حرفه‌ای)

اگه توی مرحله‌ی قبل گزینه‌ی **Generate Domain** رو زده باشید، همون دامنه‌ای که Railway ساخته رو باز کنید تا وارد پنل گرافیکی بشید.

> یوزرنیم پیش‌فرض: `admin`
> پسورد پیش‌فرض: `admin`
> در اولین ورود، پنل ازتون می‌خواد این مشخصات رو عوض کنید.

این روش برای کسایی مناسبه که پنل حرفه‌ای و سایتی می‌خوان، بدون نیاز به ورود مکرر به ترمینال برای اضافه کردن کاربر یا دیدن میزان مصرف.

### روش ۲ — خط‌فرمان داخل کانتینر (برای استفاده شخصی و ساده)

اگه فقط خودتون قراره استفاده کنید و نیازی به پنل تصویری ندارید، کافیه برید کنار همون بخش Settings، وارد تب **Console** بشید و دستور زیر رو بزنید:

```bash
dx
```

با این دستور یه منوی متنی و کاربردی براتون باز می‌شه که از همون‌جا می‌تونید کاربر اضافه کنید، حذف کنید یا وضعیت سرویس رو ببینید. همچنین می‌تونید مستقیماً از دستورات زیر هم استفاده کنید:

```bash
dx add-user client_name
dx remove-user client_name
dx status
```

### 🌍 نکته تکمیلی — تغییر ریجن سرور

ریجن پیش‌فرض Railway روی **آمریکا (US West)** هست که برای کاربرای ایرانی پینگ بالایی داره. برای بهتر شدن سرعت اتصال، از مسیر زیر ریجن رو به هلند تغییر بدید:

> **Settings → Scale → Region → Europe West (Netherlands)**
